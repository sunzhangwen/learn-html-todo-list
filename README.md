# 四象限待办清单

一款纯前端的桌面待办事项管理工具，基于艾森豪威尔矩阵（四象限法则）帮助你将任务按「重要性」与「紧急性」分类管理。单个 HTML 文件即可运行，无需安装依赖或启动服务器。

## 快速开始

1. 克隆或下载本仓库
2. 用浏览器直接打开 `index.html`
3. 在右侧面板添加你的第一个待办事项

**环境要求：**

- 现代浏览器（Chrome、Edge、Firefox、Safari 等）
- 建议屏幕分辨率 ≥ 1280×720（面向桌面端设计）

> 首次打开需要联网加载 Google Fonts（Nunito 字体）。离线环境下应用仍可正常使用，仅字体回退为系统默认无衬线字体。

## 功能介绍

### 四象限分类

任务按两个维度划分，坐标轴交汇于面板中心：

- **水平轴（X）**：左侧 = 紧急，右侧 = 不紧急
- **垂直轴（Y）**：上方 = 重要，下方 = 不重要

| 象限 | 位置 | 颜色 | 优先级标识 |
|------|------|------|-----------|
| 重要且紧急 | 左上 | 红色 `#E74C3C` | `important-urgent` |
| 重要但不紧急 | 右上 | 橙黄 `#F39C12` | `important-not-urgent` |
| 不重要但紧急 | 左下 | 蓝色 `#3498DB` | `not-important-urgent` |
| 不重要不紧急 | 右下 | 灰色 `#95A5A6` | `not-important-not-urgent` |

每条事项在对应象限内显示为一个 **22px** 的圆形标记，内部数字为创建序号（从 1 开始，按创建时间永久编号，不随排序变化）。

### 事项清单

**添加事项**（右侧面板顶部表单）：

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 事项描述 | ✅ | 单行文本，最多 200 字符 |
| 截止日期 | 可选 | 日期选择器；勾选「无截止日期」后禁用并清空 |
| 联系人 | 可选 | 标签式多选，输入后按 Enter 或逗号生成标签 |
| 优先级分类 | ✅ | 下拉选择，带颜色色块指示 |

**列表工具栏：**

- 统计信息：如「共 12 项 · 3 项已完成」
- **排序方式**：日期升/降序、手动拖拽、重要性升/降序、紧急性升/降序
- **显示已完成**：开关控制已完成事项的显示与隐藏（默认开启）

**单条事项操作：**

- 勾选复选框标记为已完成
- 点击「编辑」进入行内编辑模式
- 点击「删除」弹出确认对话框

### 可视化连线

未完成的事项会从四象限中的圆圈画一条 SVG 贝塞尔曲线，连接到右侧列表对应条目的左边缘。特性包括：

- 默认 1px 宽、半透明，颜色与象限主色一致
- 鼠标悬停在圆圈或列表条目上时，连线加粗至 2.5px 并置顶高亮
- 已完成事项保留圆圈但不显示连线
- 窗口缩放或列表滚动时自动重绘

### 悬停联动高亮

将鼠标悬停在左侧圆圈或右侧列表条目上时，会同步高亮对应的事项：

1. 连线加粗、不透明、置于最顶层
2. 圆圈发光并放大至 110%
3. 列表条目左侧显示彩色边框、背景轻微高亮
4. 其他非焦点项降低透明度，突出当前项

### 已完成事项样式

| 区域 | 已完成样式 |
|------|-----------|
| 四象限圆圈 | 半透明虚线边框，序号仍可见 |
| 列表条目 | 描述加删除线，整体透明度降低 |
| 连线 | 不显示 |

关闭「显示已完成」开关后，已完成条目和对应圆圈均会隐藏。

### 数据持久化

所有数据通过浏览器 `localStorage` 自动保存，刷新或关闭浏览器后数据不丢失。

- 存储键名：`todo-quadrant-app`
- 手动排序顺序通过 `manualOrder` 字段持久保存
- 联系人自动补全列表从所有事项的联系人字段自动提取去重

若浏览器处于隐私模式或存储空间已满，页面顶部会显示警告提示，应用仍可正常使用但数据可能无法保存。

## 使用说明

### 添加事项

1. 在「事项描述」输入框中填写任务内容
2. 按需设置截止日期（或勾选「无截止日期」）
3. 可选：在联系人输入框中输入姓名，按 **Enter** 或 **逗号** 添加标签
4. 选择优先级分类
5. 点击「添加」按钮，或在描述框中按 **Enter**

### 管理事项

- **标记完成**：勾选条目左侧的复选框
- **编辑**：点击「编辑」，修改后点「保存」；按 **Esc** 取消编辑
- **删除**：点击「删除」，在确认对话框中点「删除」
- **排序**：在工具栏下拉框中选择排序方式；选择「手动」后，可拖拽条目左侧的六点手柄调整顺序
- **筛选**：关闭「显示已完成」开关以隐藏已完成任务

### 查看四象限

添加事项后，对应圆圈会自动出现在正确的象限中。将鼠标悬停在圆圈或列表条目上，可通过连线和高亮效果快速定位关联项。

### 联系人自动补全

输入联系人时，若该姓名曾在其他事项中使用过，会自动弹出补全建议，点击即可添加。

| 按键 | 作用 |
|------|------|
| **Enter**（描述框） | 提交添加表单 |

## 数据模型

每条事项以 JSON 对象存储在 `localStorage` 数组中：

```json
{
  "id": "uuid-字符串",
  "description": "事项描述",
  "dueDate": "2025-12-31",
  "contacts": ["张三", "李四"],
  "priority": "important-urgent",
  "completed": false,
  "createdAt": 1718000000000,
  "manualOrder": 0
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识（UUID） |
| `description` | string | 事项描述 |
| `dueDate` | string \| null | 截止日期（`YYYY-MM-DD`），无截止日期时为 `null` |
| `contacts` | string[] | 联系人数组 |
| `priority` | string | 四象限优先级标识 |
| `completed` | boolean | 是否已完成 |
| `createdAt` | number | 创建时间戳（毫秒） |
| `manualOrder` | number | 手动排序序号 |

`priority` 可选值：

- `important-urgent` — 重要且紧急
- `important-not-urgent` — 重要但不紧急
- `not-important-urgent` — 不重要但紧急
- `not-important-not-urgent` — 不重要不紧急

## 常见问题

**Q：数据存在哪里？**

数据保存在当前浏览器的 `localStorage` 中，不同浏览器或设备之间不会同步。清除浏览器数据会导致待办事项丢失。

**Q：换电脑后数据还在吗？**

不在。如需跨设备使用，可将 `localStorage` 中的 `todo-quadrant-app` 键值手动导出/导入（可通过浏览器开发者工具 → Application → Local Storage 操作）。

**Q：为什么连线位置偶尔跳动？**

连线依赖 DOM 元素的实际渲染位置。窗口大小变化或列表滚动时会自动重算并重绘，属正常行为。

**Q：隐私模式下能用吗？**

可以正常使用，但数据可能无法持久保存，页面会显示相应警告。
